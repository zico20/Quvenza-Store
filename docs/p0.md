## ✅ Fix 5 — Hero Images → next/image + priority

**Problem:** Hero banner `<img>` tags were plain HTML — no AVIF/WebP conversion, no preload, no priority. Hero is the LCP element on every page.

**Solution:** Replaced both `<img>` tags in `SlideBg` with `next/image` `<Image fill priority>`.

**Changes in `src/components/home/HeroSlider.tsx`:**

```diff
+ import Image from 'next/image';

  function SlideBg({ desktop, mobile, tone, priority }) {
-   <img src={mobile}  alt="" aria-hidden="true" className="md:hidden" style={imgStyle} />
-   <img src={desktop} alt="" aria-hidden="true" className="hidden md:block" style={imgStyle} />
+   <Image src={mobile}  alt="" aria-hidden fill priority={priority} sizes="100vw" className="md:hidden object-cover" />
+   <Image src={desktop} alt="" aria-hidden fill priority={priority} sizes="100vw" className="hidden md:block object-cover" />
  }

  // Parent passes priority={idx === 0} so first slide gets LCP preload
  <SlideBg ... priority={idx === 0} />
```

**Impact:**
- Google gets AVIF/WebP formats for hero images (30–50% smaller)
- Browser preloads hero images via `<link rel="preload">` in `<head>`
- LCP score improvement expected on next Core Web Vitals run
- Gradient fallback stays in place for progressive load
